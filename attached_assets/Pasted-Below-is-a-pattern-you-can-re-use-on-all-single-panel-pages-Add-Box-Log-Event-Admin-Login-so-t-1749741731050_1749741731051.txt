Below is a pattern you can re-use on all “single-panel” pages (Add Box, Log Event, Admin Login) so the form card always fits inside the viewport without an extra scrollbar—even on short laptop screens—while still staying perfectly centered on larger monitors.

1 — Create two utility classes in static/css/custom.css
css
Copier
Modifier
/*  ---- page-level flex centre ----  */
.fullscreen-center {
  min-height: 100vh;                     /* stretch to viewport height */
  display: flex;
  align-items: center;                   /* vertical centring */
  justify-content: center;               /* horizontal centring */
  padding: 4vh 2vw;                      /* tiny breathing room */
}

/*  ---- card that never grows too tall ----  */
.fit-card {
  width: 100%;
  max-width: 540px;                      /* nice desktop width */
  max-height: 92vh;                      /* 100vh minus header/footer margins */
  overflow-y: auto;                      /* scroll *inside* card if it ever overflows */
}
Why min-height?—It centres on tall screens, yet lets the page shrink below 100 vh on devices where the mobile keyboard pushes the viewport shorter than 100 vh (Chrome/iOS quirk).

2 — Light tweak to base.html so pages can inject a body class
html
Copier
Modifier
<body class="{% block body_class %}{% endblock %}">
Now each template can do:

jinja
Copier
Modifier
{% block body_class %}fullscreen-center bg-light{% endblock %}
(Add bg-light, bg-gray-100, or your login-page class if you already set a subtle background.)

3 — Wrap the content
jinja
Copier
Modifier
<div class="fit-card card shadow rounded-4">
    <div class="card-body">
        <!-- your Add Box / Log Event / Admin form here -->
    </div>
</div>
That’s it—no row, no col-md-6 wrapper required.
Flexbox gives perfect centring; max-height: 92vh keeps the card shorter than the viewport, and any overflow scrolls inside the card, not the page.
